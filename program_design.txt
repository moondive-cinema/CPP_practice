// 디지털도어락, 인풋은 키보드로 대신함
// 문이 닫히면 자동으로 락(lock)
// 문 안쪽 단추를 누르면 언락(unlock), 다시 누르면(lock)
// 언락 상태에서 문을 열지 않고 10초 있으면 자동으로 락으로 바뀜
// 문 바깥 도어락 덮개를 열면 입력 활성화 (불이 들어옴)
// 입력 활성화 된 후 10초 간 입력이 없으면 입력비활성화 (불 꺼짐)
// 비밀번호는 4자리에서 8자리 까지 세팅 가능
// 설정된 비밀 번호를 입력하고 "*"를 누르면 언락(unlock)
// 앞에 몇 개의 번호를 누르던 마지막 입력만 비밀번호와 맞으면 언락(unlock)
// 비밀번호 오류 시 경고음 방출하고 입력 비활성화 (불이 꺼짐)
// 덮개를 다시 (닫았다) 열면 입력 활성화
// 비밀 번호 오류가 5회 나면 다른 종류의 경고음 방출하고 10분-실제는 10초 간 입력 비활성화 
// 비번 세팅은 외부 덮개를 열고 입력이 활성화 되면
// "#"을 누른 후 원래 비번(최초 비번은 000000)을 누르고 "#"을 누르면
// 새로운 비밀번호를 입력받을 준비가 되었다는 알람 방출 (불이 깜빡깜빡) 
// 새로운 비밀번호를 입력하고 "*"를 누르면 재입력 대기 (불이 깜빡깜빡)
// 다시 같은 비밀번호를 입력하고 "*"를 누르면 다시 세팅 되었다는 알람 방출 
// 재입력한 비밀번호가 틀리면 알람 방출 하고 재시도 기회 줌 (불이 깜빡깜빡)
// 재차 틀리면 틀림 알람 방출 후 입력 비활성화
// 처음 부터 다시 설정 과정을 거쳐야 함
// 완전 초기화 비밀코드(xxxxxxxxxx)도 내장
// 필요한 데이터: 비밀번호, 초기화 비밀코드, 문 잠김/풀림, 문 열림/닫힘, 입력 활성/비활성, 타이머, 램프 온/오프/블링크, 알람 타입
// 필요한 동작: lock/unlock, 먹통모드, 내부에서 열기, 외부에서 도어락 덮개 열기, 번호입력 
//           입력 활성화/비활성화 상태(with 램프, 알람), 상황에 맞는 램프, 상황에 맞는 알람
// 프로그래밍 순서 => 입출력 정의, 필요 변수 및 상수 정의, 비번 입력 알고리즘, 비번 세팅 알고리즘 
// 입력: 안쪽 열기 버튼, 바깥 쪽 도어락 덮개, 바깥 쪽 키패드
// 출력: 락/언락, 램프, 알람
